Loyalty Tiers

A feature that lets marketers build a tier program where users level up based on behavior. The biggest challenge was understanding the IA and distinguishing between concepts to name them appropriately.

The Copy in Context
Section 1 — Key Details
1 "Tier upgrade parameters" — you are choosing the inputs here, not setting the rules yet.
2 "All users start here. Once a user enters this tier, they can't downgrade." Written from the user's perspective, not the system's.
Section 2 — Tiers Qualification Criteria
3 "Qualification criteria (Based on tier upgrade parameters)" — links back to Section 1 so users know these rules apply to the parameters they set earlier.
4 Tier 2 description mirrors Tier 1's language but adds the downgrade condition, because unlike Tier 1, users can move out of Tier 2.
The Problem

The first two sections work in sequence. "Tier upgrade parameters" in "Key Details" and "Qualification criteria" under "Tier 2" (and any subsequent tiers) were labeled and described as "Tier upgrade criteria" in both the PRD and the initial designs. On the surface, that seemed fine. However, upon deeper investigation into the product's intent and the expected user journey, a distinction emerged.

Product Decoding
Field in "Key Details" was labeled
Tier upgrade criteria When it was just a picker for an event or property on which the criteria would be based
Field in "Tier 2" was labeled
Tier upgrade criteria Which was quite true to the functionality on offer there

The same label for both was incorrect and misleading for the user.

Copy Solution

Once the distinction was clear, I tried different word choices and iterations to find a pair of words that conveyed it.

Field in "Key Details" was labeled
Tier upgrade parameters
Field in "Tier 2" was labeled
Qualification criteria (Based on tier upgrade parameters):

I made a conscious choice not to call the field in "Tier 2" "Upgrade Criteria" because it could be confusing. Does it represent criteria for upgrading to Tier 3 from Tier 2, or the criteria for upgrading from Tier 1 to Tier 2? The ambiguity was real.

The Outcome

The change shipped. The team initially pushed back; the distinction was not immediately obvious. I walked them through the user journey step by step until the conceptual difference clicked. Ultimately, we left with more than a word swap: a clearer shared understanding of what the feature was doing.
